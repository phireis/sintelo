GERAÇÃO DE CÓDIGO



A geração de código é a última etapa da compilação, onde este recebe como entrada a representação intermediária do códio-fonte e produz como saída um programa alvo equivalente (AHO; SETHI; ULLMAN, 1995).



Código intermediário



Segundo Price e Toscani (2001, pg. 15) geração de código intermediário é “a transformação da árvore de derivação em um segmento de código”. Este tipo se torno muito proveitoso, pois apresenta inúmeras vantagens em relação a geração de código objeto como:

  • possibilita a otimização do código intermediário, de modo a alcançar o código objeto final mais eficiente;

  • possibilita a traduções de código fonte para diversas máquinas;

  • possibilita facilidades no processo de compilação – há abstração de problemas relativos ao ambiente.

Mas apesar das vantagem o grande problema de gerar código intermediário é que o compilador necessita fase um passo a mais, o que não acontece no código objeto, onde a tradução é direta do código fonte, sendo mais rápida a sua compilação.



Código objeto



A geração de código objeto depende da máquina hospedeira, e possui alguns requisitos, onde o código gerado deve ser correto e de alta qualidade, o código deve fazer uso efetivo dos recursos da máquina e o código gerado deve executar eficientemente.

O projetista do compilado deve considerar os 4 aspectos que segue (PRICE; TOSCANI, 2001):

  • forma do código objeto a ser gerado: linguagem absoluta, relocável ou assembly;

  • seleção das instruções de máquina: escolha da seqüência apropriada pode resultar num código mais curto e mais rápido;

  • alocação de registradores;

  • escolha da ordem de avaliação: a determinação das melhor ordem para execução das instruções e um problema insolúvel. Algumas computações requerem menos registradores para resultados intermediário.
