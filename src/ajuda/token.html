TOKEN



O analisador léxico tem a função de ler uma seqüência de caracteres que constitui um programa fonte e coletar, dessa seqüência, os tokens que constituem o programa fonte. Price e Toscani (2001) descrevem que cada token é representado internamente por três informações:

  1. Classe do token – representa o tipo de tokens reconhecidos. Exemplos: identificadores, constantes numéricas, cadeias de caracteres, palavras reservadas, operadores e separadores.

  2. Valor do token – depende da classe associada, por exemplo, para um token da classe inteira o valor do token pode ser o número inteiro representado pela constante. Mas para algumas classes de token, como as palavras reservadas, não têm valor associado.

  3. Posição do token – indica o local do texto fonte onde ocorreu o token, esta informação é utilizada para identificar o local onde ocorreu erros.

Os tokens identificados pelo analisador léxico será utilizado como entrada para o analisador sintático, segunda fase da compilação, mas o programa fonte mesmo sendo o mesmo tanto no léxico quando no sintático possui sentença de linguagens diferentes. No léxico o programa fonte é uma seqüência de palavras de uma linguagem regular, já no sintático, a seqüência de tokens constitui uma sentença de uma linguagens livre de contexto (PRICE; TOSCANI, 2001).
