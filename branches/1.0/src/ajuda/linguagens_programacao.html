LINGUAGENS DE PROGRAMAÇÃO



A linguagem é a forma mais eficaz de comunicação entre as pessoas. No computador a linguagem tem o mesmo objetivo, a diferença é que este faz a comunicação entre o programador que esta tentando resolver um determinado problema e o computador que será o solucionador. Assim sendo, linguagem de programação, segundo Price e Toscani (2001), deve unir o pensamento do desenvolvedor e a precisão requerida para o processamento pela máquina.

Atualmente as linguagens de programação que são mais utilizadas são aquelas classificadas de alto nível, que são mais próximas às linguagens naturais. Para que se tornem operacionais, os programas escritos em linguagens de alto nível devem ser traduzidos para linguagem de máquina. A conversão é realizado por sistemas especializados chamados compiladores ou interpretadores, que recebem como entrada um representação textual da solução de um problema, expresso em um linguagem fonte, e produzem uma representação do mesmo algoritmo expressa em linguagem objeto (PRICE; TOSCANI, 2001).

Evolução das Linguagens de Programação



Linguagens de programação são classificadas em cinco gerações, seguindo uma seqüência: 1ª Linguagens de máquina, 2ª linguagens simbólica (Assembly), 3ª linguagens orientadas ao usuário, 4ª linguagens orientadas à aplicação e 5ª linguagens conhecimento. As linguagens de 1ª e 2ª gerações são consideradas linguagens de baixo nível, as demais são linguagens classificadas de alto nível. Price e Toscani (2001) descrevem a evolução das linguagens conforme o que segue.

Os computadores no inicio eram programados em linguagem de máquina, ou seja em notação binária, esta linguagem permite a comunicação direta com o computador em termos de “bits”, registradores e operações de máquina bastante primitivas. O problema desta linguagem, sendo nada mais que seqüência de zeros e uns, a programação de um algoritmo mais complexo usando este tipo de linguagem é complexa, cansativa e fortemente sujeita a erros.

Já a 2ª geração onde as linguagens simbólicas ou de montagem (Assembly) estão compreendidas, foram projetadas para minimizar as dificuldades de programação em notação binária. Os códigos de operação e endereços binários formam substituídos por mnemônicos, que é uma família de códigos operacionais que realizam tarefas semelhantes no processador. Nas linguagens de montagem, a maioria das instruções são representações simbólicas de instruções de máquina, e o processamento de um programa em linguagens simbólicas requer tradução para linguagens antes de ele ser executado.

As linguagens da 3º geração são classificadas em linguagens procedimentais e linguagens declarativas. Nas linguagens procedimentais, um programa especifica um procedimento, isto é, uma seqüência de passos a serem seguidos para soluciona um problema. As instruções oferecidas por essas linguagens pertencem, em geral a três classes: instruções de entrada/saída, instruções de cálculos aritméticos ou lógicos, e instruções de controle de fluxo de execução. Os exemplos de linguagens procedimentais são: Basic, Algol, PL/I, Pascal, Ada, C e etc.

As linguagens declarativas são divididas basicamente em duas classes: funcionais, que se baseiam na teoria das funções recursivas, envolvendo, essencialmente a definição e a chamada de funções, e lógicas, que se baseiam na lógica matemática, onde um programa declara fatos (dados e relações entre eles) e cláusulas lógicas (regras de dedução), que permitirão novas verdades a partir dos fatos conhecidos. LISP é um exemplo de linguagem funcional e Prolog de linguagem lógica.

Como as linguagens de 3ª geração foram projetadas para profissionais de processamento de dados e não usuários finais, a depuração de programas escritos nestas linguagens consome tempo, e a modificação de sistemas complexos e relativamente difícil, assim as linguagens de 4ª geração foram projetadas para responder a este problema.

Nas linguagens de 4ª geração os programas escritos necessitam de menos números de linhas de código do que os programas correspondentes em linguagens de programação convencional. Muitas das linguagens de 4ª geração são dependentes de um banco de dados e de um dicionário associado, estes dicionários de dados em alguns casos, evoluíram para conter formatos de telas e de relatórios, estruturas de diálogos, mecanismos de verificação, autorizações para ler e modificar de dados e outros recursos.

As linguagens de 4ª geração permitem manipular comandos procedimentais e comandos não-procedimentais, num mesmo segmento de código; tal característica é desejável porque operações não-procedimentais aceleram e simplificam o uso da linguagem, enquanto o código procedimental entende o espaço de aplicação, dando maior flexibilidade de manipulação lógica. São exemplos de linguagem de 4ª geração: Lotus 1-2-3, Excel, SQL e FrameWork.
